LangSmith can capture traces generated by [LiveKit Agents](https://docs.livekit.io/agents/) using OpenTelemetry instrumentation. This guide shows you how to automatically capture traces from your LiveKit voice AI agents and send them to LangSmith for monitoring and analysis.

For high-level guiding principles on tracing voice agents, see [Voice tracing fundamentals](/langsmith/trace-voice-fundamentals).

<Note>
The LiveKit integration requires `langsmith[livekit]`.
</Note>

## Installation

Install the LangSmith LiveKit integration along with the LiveKit plugins you need:

<CodeGroup>

```bash pip
pip install "langsmith[livekit]" "livekit-agents[openai,silero,turn-detector]" python-dotenv
```

```bash uv
uv add "langsmith[livekit]" "livekit-agents[openai,silero,turn-detector]" python-dotenv
```

</CodeGroup>

The `langsmith[livekit]` extra installs the integration and its OpenTelemetry dependencies. Add the `livekit-agents` plugin extras for the STT, LLM, TTS, VAD, and turn-detection providers your agent uses.

## Quickstart tutorial

Follow this step-by-step tutorial to create a voice AI agent with LiveKit and LangSmith tracing. You build a complete working example by copying and pasting code snippets.

### Step 1: Set up your environment

Create a `.env` file in your project directory. The integration reads your LangSmith credentials from the standard `LANGSMITH_*` variables and exports to LangSmith for you, so no OpenTelemetry exporter variables are required:

```bash .env
LANGSMITH_API_KEY=<your-langsmith-api-key>
LANGSMITH_PROJECT=livekit-voice
LIVEKIT_URL=<your-livekit-url>
LIVEKIT_API_KEY=<your-livekit-api-key>
LIVEKIT_API_SECRET=<your-livekit-api-secret>
OPENAI_API_KEY=<your-openai-api-key>
```

### Step 2: Create your voice agent file

Create a new file called `agent.py` and add the following code. This tutorial builds it section by section so you can copy and paste each part.

#### Part 1: Import dependencies and set up tracing

`configure_livekit()` builds an OpenTelemetry tracer provider, registers the LangSmith span processor, wires it as both LiveKit's and the global tracer provider, and exports spans to LangSmith. Call it once before creating your `AgentServer`:

```python
import sys

from dotenv import load_dotenv

# Load environment variables before importing the integration
load_dotenv()

from langsmith.integrations.livekit import configure_livekit
from livekit import agents
from livekit.agents import Agent, AgentServer, AgentSession
from livekit.plugins import openai, silero
from livekit.plugins.turn_detector.multilingual import MultilingualModel

# Enable LangSmith tracing before creating agents. The project defaults to the
# LANGSMITH_PROJECT environment variable.
configure_livekit()
```

#### Part 2: Define your agent

```python
class Assistant(Agent):
    def __init__(self) -> None:
        super().__init__(
            instructions="""You are a helpful voice AI assistant.
            You eagerly assist users with their questions.
            Keep responses concise and conversational.""",
        )
```

#### Part 3: Set up the agent server

```python
server = AgentServer()

@server.rtc_session()
async def my_agent(ctx: agents.JobContext):
    # Create agent session with STT, LLM, TTS, and VAD
    session = AgentSession(
        stt="deepgram/nova-2:en",
        llm="openai/gpt-5.4-mini",
        tts=openai.TTS(model="tts-1", voice="alloy"),
        vad=silero.VAD.load(),
        turn_detection=MultilingualModel(),
    )

    # Start the session
    await session.start(
        room=ctx.room,
        agent=Assistant(),
    )

if __name__ == "__main__":
    # Run in console mode for local testing
    sys.argv = [sys.argv[0], "console"]
    agents.cli.run_app(server)
```

### Step 3: Run your agent

Run your voice agent in console mode for local testing:

```bash
python agent.py console
```

Your agent starts and connects to LiveKit. Speak through your microphone, and all traces appear automatically in LangSmith.

## How it works

`configure_livekit()` registers a span processor that translates LiveKit's `lk.*` vendor attributes into the namespaces LangSmith reads:

- Converts LiveKit span types (`user_turn`, `llm_request`, `tts_request`, `agent_turn`, `function_tool`, and so on) to LangSmith run types.
- Adds `gen_ai.prompt` and `gen_ai.completion` attributes for message visualization, including structured tool calls.
- Surfaces LiveKit's metrics (time-to-first-token, time-to-first-byte, end-to-end latency, and other `lk.*` data) as run metadata.
- Aggregates the whole-conversation transcript onto the root run, and attaches the call recording when you provide one.

It only reshapes spans it recognizes as LiveKit's; any other span (for example, a nested LangChain or LangGraph run) passes through untouched.

To manage your own `TracerProvider`, pass `set_global_provider=False` and add the returned processor to your provider:

```python
from langsmith.integrations.livekit import configure_livekit

processor = configure_livekit(set_global_provider=False)
provider.add_span_processor(processor)
```

## Advanced usage

### Trace speech-to-speech models

The previous example uses a cascade pipeline (separate STT, LLM, and TTS services). LiveKit also supports speech-to-speech models, where a single realtime model handles audio in and out. To trace one, build the session with a realtime model instead of the STT/LLM/TTS stack. The tracing setup is identical:

```python
from livekit.plugins import openai as lk_openai

session = AgentSession(llm=lk_openai.realtime.RealtimeModel(voice="marin"))
```

### Group a conversation into a thread

Pass a `thread_id_provider` to group every span in a session under a LangSmith [thread](/langsmith/threads). Provide a zero-argument callable that returns the conversation id (for example, the job id):

```python
import contextvars

_thread_id: contextvars.ContextVar[str | None] = contextvars.ContextVar(
    "thread_id", default=None
)

configure_livekit(thread_id_provider=_thread_id.get)

@server.rtc_session()
async def my_agent(ctx: agents.JobContext):
    _thread_id.set(ctx.job.id)
    ...
```

### Attach the conversation audio

To listen to a conversation alongside its transcript, record the call and attach the audio file to the root run. Record what was played to the client so the recording reflects what was actually heard, including any speech cut off by an interruption. Pass an `audio_path_provider` that returns the path to the recording, and the processor attaches it to the root run when the session ends:

```python
from pathlib import Path

_audio_path: Path | None = None

configure_livekit(audio_path_provider=lambda: _audio_path)
```

For the underlying attachment API, see [Upload files with traces](/langsmith/upload-files-with-traces).

### Custom metadata and tags

Add custom metadata to your traces using span attributes:

```python
from opentelemetry import trace

class Assistant(Agent):
    def __init__(self) -> None:
        super().__init__(
            instructions="You are a helpful assistant.",
        )

        # Get current span and add custom attributes
        span = trace.get_current_span()
        if span:
            span.set_attribute("langsmith.metadata.agent_type", "voice_assistant")
            span.set_attribute("langsmith.metadata.version", "1.0")
            span.set_attribute("langsmith.span.tags", "livekit,voice-ai,production")
```

## Troubleshooting

### Spans not appearing in LangSmith

If traces are not showing up in LangSmith:

1. **Verify credentials**: Ensure `LANGSMITH_API_KEY` is set in your `.env` file and your key has write permissions.
2. **Check setup order**: Make sure `configure_livekit()` is called before creating `AgentServer`.
3. **Check .env loading**: Ensure `load_dotenv()` is called before importing the integration and LiveKit components.

### Messages not showing correctly

If conversation messages are not displaying properly:

1. **Verify the import**: Ensure you import `configure_livekit` from `langsmith.integrations.livekit` and call it before the agent starts.
2. **Check provider**: If you manage your own `TracerProvider`, confirm the returned processor is added to it.

### Connection issues

If your agent cannot connect to LiveKit:

1. **Verify LiveKit URL**: Check `LIVEKIT_URL` is set correctly in your `.env` file.
2. **Check credentials**: Ensure `LIVEKIT_API_KEY` and `LIVEKIT_API_SECRET` are correct.
3. **Console mode**: For local testing, always use `python agent.py console`.

### Import errors

If you are getting import errors:

1. **Install dependencies**: Run the complete install command from Step 1.
2. **Check Python version**: Ensure you are using Python 3.9 or higher.
3. **Check LiveKit plugins**: Ensure you have the correct LiveKit plugins installed for your STT, LLM, and TTS providers.

### Agent not responding

If your agent connects but does not respond:

1. **Check API keys**: Verify your OpenAI API key (or other provider keys) are correct.
2. **Test services**: Ensure your STT, LLM, and TTS services are accessible.
3. **Check instructions**: Make sure your `Agent` has proper instructions.
