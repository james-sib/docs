LangSmith can capture traces generated by [Pipecat](https://pipecat.ai/) using OpenTelemetry instrumentation. This guide shows you how to automatically capture traces from your Pipecat voice AI pipelines and send them to LangSmith for monitoring and analysis.

For high-level guiding principles on tracing voice agents, see [Voice tracing fundamentals](/langsmith/trace-voice-fundamentals).

<Note>
The Pipecat integration requires `langsmith[pipecat]`.
</Note>

## Installation

Install the LangSmith Pipecat integration along with the Pipecat service extras you need:

<CodeGroup>

```bash pip
pip install "langsmith[pipecat]" "pipecat-ai[whisper,openai,local]" python-dotenv
```

```bash uv
uv add "langsmith[pipecat]" "pipecat-ai[whisper,openai,local]" python-dotenv
```

</CodeGroup>

The `langsmith[pipecat]` extra installs the integration and its OpenTelemetry dependencies. Add the `pipecat-ai` extras (`whisper`, `openai`, `local`, and so on) for the STT, LLM, TTS, and transport services your pipeline uses.

## Quickstart tutorial

Follow this step-by-step tutorial to create a voice AI agent with Pipecat and LangSmith tracing. You build a complete working example by copying and pasting code snippets.

### Step 1: Set up your environment

Create a `.env` file in your project directory. The integration reads your LangSmith credentials from the standard `LANGSMITH_*` variables and exports to LangSmith for you, so no OpenTelemetry exporter variables are required:

```bash .env
LANGSMITH_API_KEY=<your-langsmith-api-key>
LANGSMITH_PROJECT=pipecat-voice
OPENAI_API_KEY=<your-openai-api-key>
```

### Step 2: Create your voice agent file

Create a new file called `agent.py` and add the following code. This tutorial builds it section by section so you can copy and paste each part.

#### Part 1: Import dependencies

```python
import asyncio
import uuid

from dotenv import load_dotenv

# Load environment variables before importing the integration
load_dotenv()

from langsmith.integrations.pipecat import configure_pipecat
from pipecat.audio.vad.silero import SileroVADAnalyzer
from pipecat.pipeline.pipeline import Pipeline
from pipecat.pipeline.runner import PipelineRunner
from pipecat.pipeline.task import PipelineParams, PipelineTask
from pipecat.processors.aggregators.openai_llm_context import OpenAILLMContext
from pipecat.services.openai import OpenAILLMService, OpenAITTSService
from pipecat.services.whisper.stt import WhisperSTTService
from pipecat.transports.local.audio import LocalAudioTransport, LocalAudioTransportParams
```

#### Part 2: Define the main function

`configure_pipecat()` installs Pipecat's OpenTelemetry tracer provider, registers the LangSmith span processor, and exports spans to LangSmith. Call it once before building your pipeline:

```python
async def main():
    # Generate a unique conversation id for LangSmith thread grouping
    thread_id = str(uuid.uuid4())
    print(f"Starting conversation: {thread_id}")

    # Enable LangSmith tracing. Returns the span processor (used later to attach
    # audio). The project defaults to the LANGSMITH_PROJECT environment variable.
    configure_pipecat(thread_id_provider=lambda: thread_id)

    # Configure audio input/output with voice activity detection
    transport = LocalAudioTransport(
        LocalAudioTransportParams(
            audio_in_enabled=True,
            audio_out_enabled=True,
            vad_analyzer=SileroVADAnalyzer(),
        )
    )

    # Initialize AI services
    stt = WhisperSTTService()
    llm = OpenAILLMService(model="gpt-5.4-mini")
    tts = OpenAITTSService(voice="alloy")

    # Set up conversation context with system prompt
    context = OpenAILLMContext(
        messages=[
            {
                "role": "system",
                "content": "You are a helpful voice assistant. Keep responses concise and conversational."
            }
        ]
    )
    context_aggregator = llm.create_context_aggregator(context)

    # Build the processing pipeline
    pipeline = Pipeline([
        transport.input(),           # Capture microphone input
        stt,                         # Convert speech to text
        context_aggregator.user(),   # Add user message to context
        llm,                         # Generate AI response
        tts,                         # Convert response to speech
        transport.output(),          # Play through speakers
        context_aggregator.assistant(),  # Add assistant response to context
    ])

    # Create task with tracing enabled
    task = PipelineTask(
        pipeline,
        params=PipelineParams(enable_metrics=True),
        enable_tracing=True,
        enable_turn_tracking=True,
        conversation_id=thread_id,
    )

    # Run the agent
    runner = PipelineRunner()
    await runner.run(task)
```

<Note>
Enable both `enable_tracing=True` and `enable_turn_tracking=True` on the `PipelineTask`, and `enable_metrics=True` on `PipelineParams`. Turn tracking is required for tracing, and metrics drive the latency and token attributes on each span.
</Note>

#### Part 3: Add the entry point

```python
if __name__ == "__main__":
    asyncio.run(main())
```

### Step 3: Run your agent

Run your voice agent:

```bash
python agent.py
```

Speak to the agent through your microphone. All traces appear automatically in LangSmith.

## How it works

`configure_pipecat()` registers a span processor that rewrites Pipecat's OpenTelemetry spans into the attribute namespaces LangSmith reads:

- Converts Pipecat span types (`stt`, `llm`, `tts`, `turn`, `conversation`) to LangSmith run types.
- Adds `gen_ai.prompt` and `gen_ai.completion` attributes for message visualization.
- Renders the whole-conversation transcript onto the root run.
- Attaches the conversation audio recording when you wire an audio buffer (see [Record and attach audio](#record-and-attach-audio)).

It only reshapes spans it recognizes as Pipecat's; any other span (for example, a nested LangChain or LangGraph run) passes through untouched.

To manage your own `TracerProvider` instead of letting `configure_pipecat()` install one, add `PipecatLangSmithSpanProcessor` to your provider directly:

```python
from langsmith.integrations.pipecat import PipecatLangSmithSpanProcessor

provider.add_span_processor(PipecatLangSmithSpanProcessor())
```

## Advanced usage

### Trace a nested LangGraph agent

Instead of a stock LLM service, you can use an in-process [LangChain or LangGraph](/oss/langgraph/overview) agent as the LLM stage of your pipeline. With a few adjustments, the agent's model and tool runs nest inside Pipecat's `llm` span so the whole conversation stays a single trace.

Three things make this work:

1. **Set `LANGSMITH_TRACING_MODE=otel`.** This makes the LangSmith SDK emit your LangChain or LangGraph runs as OpenTelemetry spans through the same provider, so they nest under Pipecat's `llm` span instead of forming separate top-level traces.
2. **Use a traced LLM service.** A bare `FrameProcessor` produces no `llm` span for the graph's runs to nest under. Subclass a traced service such as `OpenAILLMService` and run your graph from the traced context handler so its runs land inside the `llm` span.
3. **Set `llm_span_kind="chain"`.** With the graph nested inside, Pipecat's `llm` span no longer performs the inference itself: the graph's own model nodes are the real LLM runs. Marking the wrapper a `chain` avoids an LLM run nested inside another LLM run.

```python
configure_pipecat(llm_span_kind="chain", thread_id_provider=lambda: thread_id)
```

The resulting trace looks like this:

```text
conversation                        # root: whole transcript + audio recording
└── turn × N
    ├── stt
    ├── llm                         # chain (orchestrates the graph)
    │   ├── model                   # ChatOpenAI (may emit tool calls)
    │   ├── tools: lookup_weather   # tool run
    │   └── model                   # final answer (spoken)
    └── tts
```

<Note>
Only the final answer is spoken, but the tool-call exchange must be persisted back into Pipecat's `LLMContext` (as OpenAI-format messages) or the model loses its tool history on later turns.
</Note>

### Record and attach audio

Record the conversation and attach the audio to the root run so you can listen to it alongside the transcript. The integration uses Pipecat's own [`AudioBufferProcessor`](https://docs.pipecat.ai/server/utilities/audio/audio-recording), which records **what was heard**: placed after `transport.output()`, it captures the bot audio as actually played (reached only after a barge-in truncates the agent mid-sentence) plus the user's microphone.

Create the buffer, wire it to the span processor with `attach_audio_buffer`, add it to the pipeline after `transport.output()`, and start it once the session is running:

```python
from pipecat.processors.audio.audio_buffer_processor import AudioBufferProcessor

span_processor = configure_pipecat(thread_id_provider=lambda: thread_id)

# Stereo recording: user on the left channel, bot on the right. A non-zero
# buffer_size streams audio in periodically so it is accumulated before the
# conversation span is exported.
audiobuffer = AudioBufferProcessor(num_channels=2, buffer_size=32_000)
span_processor.attach_audio_buffer(audiobuffer, conversation_id=thread_id)

pipeline = Pipeline([
    transport.input(),
    stt,
    context_aggregator.user(),
    llm,
    tts,
    transport.output(),
    audiobuffer,                     # after output(): records what was heard
    context_aggregator.assistant(),
])

# Start recording once the session is live
await audiobuffer.start_recording()
```

The processor accumulates the merged stereo PCM, encodes it to a WAV, and attaches it to the conversation root span when that span ends. For the underlying attachment API, see [Upload files with traces](/langsmith/upload-files-with-traces).

### Custom metadata and tags

Add custom metadata to your traces using span attributes:

```python
from opentelemetry import trace

tracer = trace.get_tracer(__name__)

async def run_voice_session():
    with tracer.start_as_current_span("voice_conversation") as span:
        # Add custom metadata
        span.set_attribute("langsmith.metadata.session_type", "voice_assistant")
        span.set_attribute("langsmith.metadata.user_id", "user_123")
        span.set_attribute("langsmith.span.tags", "pipecat,voice-ai,stt-llm-tts")

        # Your Pipecat pipeline code here
        task = PipelineTask(pipeline, enable_tracing=True)
        await task.queue_frames([TextFrame("Hello")])
```

## Troubleshooting

### Spans not appearing in LangSmith

If traces are not showing up in LangSmith:

1. **Verify credentials**: Ensure `LANGSMITH_API_KEY` is set in your `.env` file and your key has write permissions.
2. **Verify the call**: Make sure `configure_pipecat()` runs before the pipeline starts.
3. **Check .env loading**: Ensure `load_dotenv()` is called before importing the integration and Pipecat components.

### Messages not showing correctly

If conversation messages are not displaying properly:

1. **Verify thread ID**: Ensure you set a unique `conversation_id` on `PipelineTask`.
2. **Enable turn tracking**: Make sure `enable_turn_tracking=True` is set on `PipelineTask`.

### Audio not working

If your microphone or speakers are not working:

1. **Check permissions**: Ensure your terminal or IDE has microphone access.
2. **Test audio devices**: Verify your microphone and speakers work in other applications.
3. **VAD settings**: Try adjusting `SileroVADAnalyzer()` settings if speech is not being detected.
4. **Check services**: Ensure your OpenAI API key is valid and has access to Whisper and TTS.

### Audio recording not attached

If the conversation audio is missing from the root run:

1. **Buffer placement**: Ensure the `AudioBufferProcessor` is placed after `transport.output()` in the pipeline.
2. **Recording started**: Confirm you called `await audio_buffer.start_recording()` once the session is running.
3. **Buffer size**: Set a non-zero `buffer_size` so audio streams in before the conversation span exports.

### Import errors

If you are getting import errors:

1. **Install dependencies**: Run `pip install "langsmith[pipecat]" "pipecat-ai[whisper,openai,local]" python-dotenv`.
2. **Check Python version**: Ensure you are using Python 3.9 or higher.
