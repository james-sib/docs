Online evaluators call LLMs and incur cost. LangSmith surfaces weekly evaluator spend across your workspace and lets workspace admins cap spend on each evaluator attachment, so a runaway evaluator does not burn through your budget unattended.

This page covers:

- Viewing weekly spend per evaluator and per attached resource
- Setting a workspace-wide default spend limit
- Overriding the default for an individual evaluator attachment
- What happens when an attachment hits its limit

Spend totals are computed over a rolling 7-day window. The UI labels this window as **/ week**.

## View evaluator spend

Three surfaces show evaluator spend.

### Evaluators page dashboard

The top of the **Evaluators** page (left sidebar > **Evaluators**) shows a weekly view across the workspace:

- **Daily evaluator spend**: Stacked bar chart of spend per day. Toggle between **Evaluator** and **Project / Dataset** breakdowns.
- **Evaluator spend this week**: Total USD spend across all evaluators, with the change versus the previous week.
- **Evaluator runs this week**: Total run count across all evaluators, with the change versus the previous week.
- **Weekly evaluator spend limit monitoring**: Sortable list of top spenders with a per-attachment progress bar against its `$ spent / $ limit`. A header chip surfaces the count of attachments that have hit their limit.

Use the **Prev week** and **Next week** controls in the page header to move the 7-day window.

### Evaluators table

The Evaluators table on the same page includes:

- **Spend**: Total LLM cost for the evaluator across all attached resources over the last 7 days. Sortable. Evaluators that do not call an LLM (for example, code evaluators) show no value.
- **Spend Status**: One of the following:
    - **Under limits**: At least one attachment has a limit, and none are at the cap.
    - **N limit hit**: One or more attachments have reached their limit and are paused. The number reflects how many attachments are paused.
    - **Unlimited**: No attachment has a limit set.

### Resources tab on an evaluator

Open an evaluator and select the **Resources** tab to see per-attachment spend and limits:

- **Spend limit**: Effective weekly limit for that attachment, either the workspace default or a per-attachment override.
- **Current spend**: Progress bar showing spend against the limit over the rolling 7-day window.

For attachment management, refer to [Manage evaluators](/langsmith/evaluators).

The **resource details** view (a project or dataset) has an **Evaluators** tab that mirrors these widgets scoped to that resource.

## Set a workspace default spend limit

Workspace admins can set a default weekly spend limit that applies to every evaluator attachment in the workspace.

<Note>Setting and editing the workspace default requires the `organization:manage` [permission](/langsmith/rbac).</Note>

1. Navigate to **Settings** > **Usage Configuration** and open the workspace.
1. Find the **Evaluator spend limit (per project/dataset attachment)** field.
1. Enter a USD amount. The unit is `/ week`. Leave blank for no limit.
1. Click **Save**.

If no workspace default is set, evaluator attachments are unlimited unless a per-attachment override is configured. Clearing the workspace default removes the cap from every attachment that inherits it.

If you change the workspace default while attachments already have custom overrides, LangSmith asks whether to overwrite those overrides with the new default. Review the listed attachments before confirming.

## Override the default for an attachment

Workspace admins can override the workspace default for a specific evaluator attachment.

1. Navigate to **Evaluators** in the left sidebar and open the evaluator.
1. Select the **Resources** tab and edit the attachment.
1. Expand **Advanced**.
1. In the **Spend limit** field, set a custom USD amount. The unit is `/ week`.
1. Save the attachment.

The hint text below the field shows whether the current value is the workspace default or a custom limit. To revert an override back to the workspace default, click **Reset to workspace default**.

Members without `organization:manage` see the effective limit but cannot modify it. The read-only view shows one of:

- `Unlimited / week (workspace default)`
- `$<amount> / week (workspace default)`
- `$<amount> / week (custom limit)`

## What happens when a limit is reached

When an attachment's weekly spend reaches its effective limit:

- LangSmith stops enqueueing new evaluator batches for that attachment.
- The Evaluators table **Spend Status** column shows **N limit hit**, and the Weekly evaluator spend limit monitoring widget surfaces the affected attachments.
- Skipped runs are not backfilled. Evaluation resumes automatically on new runs once the rolling 7-day spend falls below the limit.

Spend is recorded after evaluator runs complete, so an attachment can briefly exceed its limit by a small amount while in-flight batches finalize.

## Model pricing requirements

Spend tracking and limits depend on per-model pricing stored in LangSmith. Confirm that every model used by your evaluators has pricing configured in [Model configurations](/langsmith/model-configurations) and that runs surface cost via [Cost tracking](/langsmith/cost-tracking).

If a model has no pricing configured, its runs are not counted toward spend, and a spend limit cannot enforce against it.
