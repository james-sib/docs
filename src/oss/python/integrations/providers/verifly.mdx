[Verifly](https://verifly.email) is an agent-native email verification API. Given an
email address, it checks deliverability—syntax, domain, MX records, SMTP, plus
disposable, role-account, catch-all, and free-provider signals—and returns a clear
verdict (`deliverable`, `undeliverable`, or `risky`), a reason, a confidence score, and
a send/reject recommendation. The `langchain-verifly` package wraps this API as a
LangChain tool so an agent can validate a lead or signup before sending mail.

## Installation and setup

<CodeGroup>
```bash pip
pip install langchain-verifly
```

```bash uv
uv add langchain-verifly
```
</CodeGroup>

Get an API key (with free credits) from [verifly.email](https://verifly.email) and set
it as an environment variable:

```bash
export VERIFLY_API_KEY="your-api-key"
```

## Tools

The package provides `VeriflyEmailVerifier`, a LangChain tool with both sync and async
support:

```python
from langchain_verifly import VeriflyEmailVerifier

verifier = VeriflyEmailVerifier()  # reads VERIFLY_API_KEY from the environment

verifier.invoke({"email": "lead@example.com"})
# {
#     "email": "lead@example.com",
#     "result": "deliverable",
#     "reason": "Mailbox exists",
#     "confidence": 95,
#     "is_valid": True,
#     "recommendation": "send",
#     "did_you_mean": None,
#     "details": {...},
# }
```

The tool can be passed directly to an agent so it can verify addresses on its own—for
example, scrubbing a lead list, validating a user-supplied email at signup, or
confirming a contact before drafting outreach.
